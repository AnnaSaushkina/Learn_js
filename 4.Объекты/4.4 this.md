### `this` — это ссылка на контекст выполнения функции.  

### Определяется в момент вызова, а не объявления.

| Контекст вызова | Значение `this` |
|-----------------|-----------------|
| **Метод объекта** | Сам объект |
| **Простой вызов функции** | `undefined` (strict) / `window` (non-strict) |
| **Стрелочная функция** | Берет из внешней области |
| **Конструктор** | Новый создаваемый объект |
| **Обработчик события** | Элемент, вызвавший событие |


### Потеря контекста
Возникает, когда функция теряет связь с исходным объектом.  
**Частая проблема в колбэках.**


### Стрелочные функции 
- **Не имеют своего `this`**  
- **Берут контекст из внешней области**  
- **Не подходят как методы объекта** (если нужен доступ к объекту через `this`)


### Методы привязки контекста
- **`call()`** — вызывает функцию с указанным `this` и аргументами
- **`apply()`** — аналогично `call`, но аргументы передаются массивом
- **`bind()`** — создает новую функцию с навсегда привязанным контекстом



1. **`this` динамическое** — зависит от вызова
2.  **Стрелочные функции** не имеют своего `this`
3. **Потеря контекста** — частая проблема в колбэках
4. **`bind()` создает неизменяемую привязку**


### когда что использовать
```javascript
// Методы объектов — обычные функции
const obj = {
  method() { /* this = obj */ }
};

// Колбэки — стрелочные функции  
setTimeout(() => { /* this из внешней области */ });

// Явная привязка — bind/call/apply
const boundFunc = obj.method.bind(obj);
```

`this` — это контекст вызова, а не объявления. Стрелочные функции для сохранения контекста в колбэках и `bind` для явной привязки.
