## Методы массивов

Мутируют (меняют исходный массив):

- `push`, `pop` (конец)
- `shift`, `unshift` (начало)
- `splice` (вырезать/вставить/заменить кусок)
- `sort`, `reverse`
- `fill`, `copyWithin`

Не мутируют (возвращают новый массив/значение):

- `slice`, `concat`
- `map`, `filter`
- `find`, `findIndex`, `some`, `every` (возвращают не массив, а элемент/индекс/true/false)
- `reduce`, `reduceRight`
- `flat`, `flatMap`
- `includes`, `indexOf`, `lastIndexOf` (поиск)

## 2) Редактирование списка (вставка/удаление/замена)

- Удалить 1 элемент по индексу: `arr.splice(i, 1)`
- Вставить в позицию: `arr.splice(i, 0, newItem)`
- Заменить: `arr.splice(i, 1, newItem)`
- Удалять через `delete arr[i]` почти никогда не надо: останется “дырка” (элемент станет `empty`), а длина не уменьшится.

## 3) (filter → map → sort → reduce)

Это самый частый шаблон для массивов объектов:

- `filter` — оставить только нужные
- `map` — превратить в нужный вид (например, взять поля)
- `sort` — упорядочить (если нужно)
- `reduce` — получить одно итоговое значение (сумма, объект группировки, Map и т.п.)

## 4) Поиск: “нужен элемент / индекс / просто проверить”

- Нужен элемент: `find` (первый подходящий) или `filter` (все подходящие)
- Нужен индекс: `findIndex`
- Проверка “хотя бы один?”: `some`
- Проверка “все подходят?”: `every`
- Проверка “есть значение в массиве”: `includes` (важный плюс: нормально находит `NaN`)
