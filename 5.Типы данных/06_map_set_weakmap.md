# Map / Set / WeakMap / WeakSet

## Map vs обычный объект

| | Объект | Map |
|---|---|---|
| Ключи | только строки | любой тип |
| Размер | нет | `.size` |
| Порядок | не гарантирован | гарантирован |

```ts
const map = new Map();
map.set("count", 0);
map.set("count", map.get("count") + 1); // перезапишет — не создаст новый ключ
map.get("count"); // 1
map.size; // 1
```

## Set — коллекция без дубликатов

```ts
const arr = [1, 2, 2, 3, 3];
const unique = new Set(arr); // Set { 1, 2, 3 }
```

**Важно:** для объектов Set сравнивает по **ссылке**, не по значению:

```ts
const set = new Set();
const user = { name: "Аня" };
set.add(user);
set.add(user);            // игнорируется — та же ссылка
set.add({ name: "Аня" }); // добавится — другая ссылка в памяти
set.size; // 2
```

## WeakMap и WeakSet

Отличие от Map/Set — **не мешают сборщику мусора**. Когда объект-ключ удаляется из программы — данные удаляются автоматически.

Ограничения:
- Ключом может быть только объект
- Нет `.size`, нет перебора

```ts
// Map — пароль остаётся в памяти даже после удаления пользователя
// WeakMap — пароль удаляется вместе с пользователем
const passwords = new WeakMap();
let user = new User("Аня", "пароль123");
user = null; // пароль тоже удалится из памяти
```

**WeakSet** — то же самое что WeakMap, но хранит просто набор объектов без значений.
